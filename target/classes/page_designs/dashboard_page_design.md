# 仪表盘（首页）设计

## 1. 页面概述
- **页面URL**: `/admin/dashboard`
- **功能描述**: 系统首页，用于展示电商平台的核心数据和待处理事项，帮助管理员快速了解系统运行状况

## 2. 页面布局
- **顶部导航栏**: 系统名称、用户信息、通知中心、退出登录
- **左侧菜单栏**: 系统功能菜单
- **主内容区**: 采用卡片式布局，包含以下模块：
  - 数据概览卡片组
  - 销售趋势图
  - 最新订单列表
  - 待处理事项提醒

## 3. 数据概览模块

### 3.1 模块概述
- **功能描述**: 展示今日核心数据，包括订单数、销售额、访客数等
- **布局**: 4个并列卡片，每行2个

### 3.2 卡片设计

#### 3.2.1 今日订单卡片
| 元素名 | 类型 | 内容 | 交互 | 数据来源 |
|-------|------|------|------|----------|
| 标题 | 文本 | 今日订单 | 无 | 无 |
| 数值 | 文本 | 显示今日订单数量 | 点击跳转到订单列表页 | order_orders.created_at |
| 环比增长 | 文本 | 显示与昨日相比的增长率 | 无 | 无 |
| 图标 | 图标 | 订单图标 | 无 | 无 |
| 状态标签 | 标签 | 待处理订单数量 | 点击跳转到待处理订单列表 | order_orders.order_status=0或1 |

#### 3.2.2 今日销售额卡片
| 元素名 | 类型 | 内容 | 交互 | 数据来源 |
|-------|------|------|------|----------|
| 标题 | 文本 | 今日销售额 | 无 | 无 |
| 数值 | 文本 | 显示今日实际支付金额总和 | 点击跳转到销售统计页 | order_orders.actual_amount |
| 环比增长 | 文本 | 显示与昨日相比的增长率 | 无 | 无 |
| 图标 | 图标 | 金额图标 | 无 | 无 |
| 状态标签 | 标签 | 今日退款金额 | 点击跳转到退款订单列表 | order_orders.order_status=5或6 |

#### 3.2.3 今日访客数卡片
| 元素名 | 类型 | 内容 | 交互 | 数据来源 |
|-------|------|------|------|----------|
| 标题 | 文本 | 今日访客数 | 无 | 无 |
| 数值 | 文本 | 显示今日网站访客数量 | 点击跳转到访客统计页 | 系统日志或第三方统计 |
| 环比增长 | 文本 | 显示与昨日相比的增长率 | 无 | 无 |
| 图标 | 图标 | 访客图标 | 无 | 无 |
| 状态标签 | 标签 | 今日新注册用户数 | 点击跳转到用户列表页 | customer_customers.register_time |

#### 3.2.4 商品库存预警卡片
| 元素名 | 类型 | 内容 | 交互 | 数据来源 |
|-------|------|------|------|----------|
| 标题 | 文本 | 库存预警 | 无 | 无 |
| 数值 | 文本 | 显示库存不足的商品数量 | 点击跳转到库存管理页 | product_products.stock < 预警值 |
| 环比变化 | 文本 | 显示与昨日相比的变化量 | 无 | 无 |
| 图标 | 图标 | 库存图标 | 无 | 无 |
| 状态标签 | 标签 | 今日新增商品数 | 点击跳转到商品列表页 | product_products.created_at |

### 3.3 交互设计
- 卡片支持点击跳转至对应详情页面
- 数值显示支持悬停查看具体数据
- 环比增长支持颜色标识（正数绿色，负数红色）
- 自动刷新数据，默认每5分钟刷新一次

## 4. 销售趋势图模块

### 4.1 模块概述
- **功能描述**: 展示销售数据随时间的变化趋势
- **布局**: 独立卡片，包含图表和筛选条件

### 4.2 筛选条件设计
| 字段名 | 类型 | 必填 | 说明 | 选项 |
|-------|------|------|------|------|
| 时间范围 | 单选按钮组 | 否 | 选择数据展示的时间范围 | 今日、昨日、近7天、近30天、自定义 |
| 数据类型 | 单选按钮组 | 否 | 选择图表展示的数据类型 | 订单数、销售额、访客数 |
| 图表类型 | 单选按钮组 | 否 | 选择图表展示类型 | 折线图、柱状图 |
| 自定义时间范围 | 日期范围选择器 | 否 | 当时间范围选择自定义时显示 | 开始日期、结束日期 |

### 4.3 图表设计
- **图表类型**: 支持折线图和柱状图切换
- **X轴**: 时间（小时、天、周）
- **Y轴**: 数值（订单数、销售额、访客数）
- **图例**: 显示数据类型
- **交互**: 支持悬停查看具体数值，支持缩放和平移

### 4.4 数据来源
- **订单数**: order_orders表，按时间分组统计
- **销售额**: order_orders表，按时间分组统计actual_amount总和
- **访客数**: 系统日志或第三方统计数据

### 4.5 交互设计
- 支持切换时间范围查看不同时间段的数据
- 支持切换数据类型查看不同维度的数据
- 支持切换图表类型
- 支持自定义时间范围
- 图表支持悬停查看具体数据
- 支持点击图表区域跳转到对应的数据详情页

## 5. 最新订单列表模块

### 5.1 模块概述
- **功能描述**: 展示最近的订单信息，方便管理员快速了解订单情况
- **布局**: 独立卡片，包含表格和操作按钮

### 5.2 表格设计
| 列名 | 类型 | 操作 | 数据来源 |
|------|------|------|----------|
| 订单编号 | 文本 | 点击查看订单详情 | order_orders.order_no |
| 客户名称 | 文本 | 点击查看客户详情 | order_orders.customer_name |
| 订单金额 | 文本 | 无 | order_orders.actual_amount |
| 订单状态 | 标签 | 无 | order_orders.order_status |
| 支付方式 | 文本 | 无 | order_orders.pay_type |
| 下单时间 | 时间 | 无 | order_orders.created_at |
| 操作 | 按钮组 | 查看详情、处理订单 | 无 |

### 5.3 操作按钮设计
| 按钮名 | 类型 | 功能 | 权限 |
|-------|------|------|------|
| 查看全部 | 普通按钮 | 跳转到订单列表页 | 所有角色 |
| 刷新 | 图标按钮 | 刷新最新订单列表 | 所有角色 |

### 5.4 交互设计
- 表格支持分页，默认显示10条记录
- 订单编号支持点击跳转到订单详情页
- 客户名称支持点击跳转到客户详情页
- 订单状态使用不同颜色标识：
  - 待付款：橙色
  - 待发货：蓝色
  - 待收货：绿色
  - 已完成：灰色
  - 已取消：红色
  - 已退款：紫色
  - 已退货：深红色
- 支持按订单状态筛选
- 自动刷新数据，默认每2分钟刷新一次

## 6. 待处理事项提醒模块

### 6.1 模块概述
- **功能描述**: 展示需要管理员处理的事项，提醒管理员及时处理
- **布局**: 独立卡片，包含待处理事项列表和操作按钮

### 6.2 待处理事项列表
| 事项类型 | 数量 | 描述 | 操作 | 数据来源 |
|---------|------|------|------|----------|
| 待付款订单 | 数字 | 显示待付款订单数量 | 点击跳转到待付款订单列表 | order_orders.order_status=0 |
| 待发货订单 | 数字 | 显示待发货订单数量 | 点击跳转到待发货订单列表 | order_orders.order_status=1 |
| 待收货订单 | 数字 | 显示待收货订单数量 | 点击跳转到待收货订单列表 | order_orders.order_status=2 |
| 退款申请 | 数字 | 显示待处理退款申请数量 | 点击跳转到退款申请列表 | order_orders.order_status=5 |
| 退货申请 | 数字 | 显示待处理退货申请数量 | 点击跳转到退货申请列表 | order_orders.order_status=6 |
| 商品审核 | 数字 | 显示待审核商品数量 | 点击跳转到商品审核列表 | product_products（假设状态） |
| 投诉处理 | 数字 | 显示待处理投诉数量 | 点击跳转到投诉处理列表 | 系统消息表（假设） |
| 系统通知 | 数字 | 显示未读系统通知数量 | 点击跳转到通知列表 | sys_messages.is_read=0 |

### 6.3 操作按钮设计
| 按钮名 | 类型 | 功能 | 权限 |
|-------|------|------|------|
| 查看全部 | 普通按钮 | 跳转到待处理事项汇总页 | 所有角色 |
| 标记已读 | 普通按钮 | 标记所有待处理事项为已读 | 所有角色 |
| 刷新 | 图标按钮 | 刷新待处理事项列表 | 所有角色 |

### 6.4 交互设计
- 待处理事项数量使用红色角标显示
- 支持点击事项类型跳转到对应处理页面
- 支持标记待处理事项为已读
- 自动刷新数据，默认每分钟刷新一次

## 7. 通用设计规范

### 7.1 样式规范
- 使用Ant Design/Element UI组件库
- 统一的颜色主题和字体
- 响应式布局，适配不同屏幕尺寸
- 卡片式设计，保持页面整洁有序
- 关键数据突出显示，使用较大字体和醒目的颜色

### 7.2 交互规范
- 表单验证实时反馈
- 操作成功/失败提示
- 确认弹窗（删除、批量操作等）
- 加载状态提示
- 支持键盘快捷键操作
- 数据自动刷新，可配置刷新频率

### 7.3 数据规范
- 所有数据操作都需记录操作日志
- 支持数据的导入导出
- 支持数据的批量操作
- 数据展示支持分页和筛选

## 8. 技术实现建议

### 8.1 前端技术栈
- React/Vue
- Ant Design/Element UI
- ECharts（图表库）
- Axios
- React Router/Vue Router
- Day.js（时间处理）

### 8.2 后端技术栈
- Spring Boot
- MyBatis-Plus
- MySQL
- Redis（缓存热点数据）
- Elasticsearch（日志和统计数据）

### 8.3 性能优化
- 仪表盘数据使用缓存
- 图表数据预计算
- 分页加载列表数据
- 延迟加载非关键数据
- 数据自动刷新采用长轮询或WebSocket

### 8.4 安全考虑
- API接口权限控制
- 敏感数据加密传输
- 防止SQL注入
- 防止XSS攻击
- 限流保护，防止恶意请求

## 10. 总结

仪表盘（首页）是电商后台管理系统的核心页面，用于展示系统的核心数据和待处理事项。本设计方案基于MySQL数据库表结构，结合了现代前端技术和最佳实践，实现了数据概览、销售趋势图、最新订单列表和待处理事项提醒四个主要功能模块。

设计方案考虑了用户体验、性能优化和安全考虑，并提供了后续迭代建议，为电商后台管理系统的仪表盘功能提供了完整的解决方案。
