# 登录页面设计

## 1. 页面概述
- **页面URL**: `/admin/login`
- **功能描述**: 系统登录入口，用于管理员和员工登录后台管理系统
- **适用角色**: 所有系统用户

## 2. 页面布局
- **整体布局**: 居中对齐，采用卡片式设计
- **顶部**: 系统Logo和标题
- **中部**: 登录表单（用户名、密码、验证码输入框及登录按钮）
- **底部**: 忘记密码链接、版权信息

## 3. 表单设计

### 3.1 登录表单
| 字段名 | 类型 | 必填 | 说明 | 关联字段 | 验证规则 |
|-------|------|------|------|----------|----------|
| 用户名 | 输入框 | 是 | 登录用户名 | sys_users.username | 非空验证，最大长度50字符 |
| 密码 | 密码输入框 | 是 | 登录密码 | sys_users.password | 非空验证，支持密码可见性切换 |
| 验证码 | 输入框+图片 | 是 | 防止机器人登录 | 无 | 非空验证，4位字符，区分大小写 |

### 3.2 表单元素详细设计

#### 3.2.1 用户名输入框
- **类型**: 文本输入框
- **占位符**: 请输入用户名
- **图标**: 用户头像图标
- **交互**: 聚焦时边框高亮，失焦时验证
- **错误提示**: 用户名不能为空

#### 3.2.2 密码输入框
- **类型**: 密码输入框
- **占位符**: 请输入密码
- **图标**: 锁图标
- **交互**: 聚焦时边框高亮，失焦时验证，支持点击切换密码可见性
- **错误提示**: 密码不能为空

#### 3.2.3 验证码区域
- **布局**: 左侧输入框，右侧验证码图片
- **验证码输入框**: 
  - 类型: 文本输入框
  - 占位符: 请输入验证码
  - 最大长度: 4位
- **验证码图片**: 
  - 类型: 图片
  - 交互: 点击可刷新验证码
  - 大小: 120px × 40px
- **错误提示**: 验证码不能为空，验证码错误

#### 3.2.4 登录按钮
- **类型**: 主按钮
- **文本**: 登录
- **尺寸**: 宽100%
- **交互**: 点击触发登录请求，加载状态显示加载动画
- **禁用状态**: 表单验证不通过时禁用

#### 3.2.5 忘记密码链接
- **类型**: 链接文本
- **文本**: 忘记密码？
- **位置**: 登录按钮下方，靠右对齐
- **交互**: 点击跳转到忘记密码页面

### 3.3 表单验证规则
| 字段 | 验证规则 | 错误提示信息 |
|------|----------|--------------|
| 用户名 | 非空 | 请输入用户名 |
| 密码 | 非空 | 请输入密码 |
| 验证码 | 非空且等于图片验证码 | 请输入验证码<br>验证码错误 |

## 4. 页面元素设计

### 4.1 系统Logo
- **位置**: 登录卡片顶部中央
- **尺寸**: 80px × 80px
- **类型**: 图片或SVG图标

### 4.2 系统标题
- **位置**: Logo下方
- **文本**: 电商后台管理系统
- **字体**: 加粗，20px
- **颜色**: 主题色

### 4.3 版权信息
- **位置**: 页面底部
- **文本**: © 2024 电商后台管理系统 版权所有
- **字体**: 12px，灰色

## 5. 交互设计

### 5.1 表单交互
- **实时验证**: 输入框失焦时进行验证，显示错误提示
- **密码可见性**: 密码输入框右侧有切换图标，点击可显示/隐藏密码
- **验证码刷新**: 点击验证码图片可刷新验证码
- **回车键提交**: 输入框聚焦时按下回车键可提交表单

### 5.2 登录流程
1. 用户输入用户名、密码和验证码
2. 点击登录按钮或按下回车键
3. 系统验证表单数据
4. 验证通过后，发送登录请求到后端
5. 登录成功：跳转到系统首页（仪表盘）
6. 登录失败：显示错误提示信息（如用户名或密码错误、验证码错误等）

### 5.3 错误处理
- **用户名或密码错误**: 显示"用户名或密码错误，请重试"
- **验证码错误**: 显示"验证码错误，请重试"
- **网络错误**: 显示"网络连接失败，请检查网络设置"
- **服务器错误**: 显示"系统繁忙，请稍后重试"

### 5.4 加载状态
- **登录按钮**: 点击后显示加载动画，按钮文本变为"登录中..."
- **验证码刷新**: 点击后显示加载动画，刷新完成后显示新验证码

## 7. 安全设计

### 7.1 前端安全
- **密码加密**: 密码在前端不加密，通过HTTPS传输
- **防止XSS**: 输入内容进行转义处理
- **防止CSRF**: 登录请求携带CSRF令牌
- **验证码**: 防止暴力破解
- **密码可见性**: 支持切换，但默认隐藏

### 7.2 后端安全
- **密码存储**: 数据库中密码使用BCrypt等算法加密存储
- **登录限制**: 支持IP或用户名的登录失败次数限制
- **会话管理**: 登录成功后生成安全的会话ID，设置合理的过期时间
- **权限验证**: 登录后验证用户权限

## 8. 技术实现建议

### 8.1 前端技术栈
- **框架**: React/Vue
- **UI组件库**: Ant Design/Element UI
- **HTTP客户端**: Axios
- **表单验证**: 内置表单验证或第三方库（如Yup、VeeValidate）

### 8.2 后端技术栈
- **框架**: Spring Boot
- **安全框架**: Spring Security
- **密码加密**: BCrypt
- **验证码生成**: Kaptcha或自定义实现

### 8.3 性能优化
- **资源压缩**: 图片、CSS、JS资源压缩
- **CDN加速**: 静态资源使用CDN
- **缓存**: 验证码图片缓存（合理时间）
- **异步加载**: 非关键资源异步加载

## 10. 后续迭代建议

### 10.1 功能扩展
- **记住我功能**: 支持记住用户名

## 11. 总结

登录页面是电商后台管理系统的重要入口，设计时需要考虑安全性、易用性和美观性。本设计方案基于MySQL数据库的sys_users表，结合了现代前端技术和最佳实践，实现了用户名/密码输入、验证码、登录按钮和忘记密码链接等核心功能。

设计方案考虑了用户体验、安全设计和性能优化，并提供了后续迭代建议，为电商后台管理系统的登录功能提供了完整的解决方案。
